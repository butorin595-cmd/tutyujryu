# Цифровой строительный паспорт

Этот проект представляет собой учебную реализацию цифрового строительного паспорта на основе стека React + Express. Клиентское приложение позволяет просматривать и управлять файлами (документы, фотографии, таблицы), выбирать фон приложения и работать с карточкой строительного проекта.

## Возможности

- **Карточка проекта** — отображает основную информацию об объекте (название, адрес, даты начала и окончания, статус и ответственное лицо) и позволяет редактировать эти данные через интерфейс.
- **Выбор и загрузка фонового изображения** — фотографии для фона хранятся на сервере. Пользователь может выбрать любой существующий фон или загрузить новый.
- **Работа с файлами** — разделы «Документы», «Фото» и «Таблицы» позволяют загружать и удалять файлы. Доступны предпросмотр PDF, просмотр таблиц XLSX/CSV, лёгкий просмотр фотографий (lightbox) и возможность скачивания.

### Новые улучшения

- **Уведомления** — все действия (загрузка, сохранение, ошибки) сопровождаются всплывающими уведомлениями (toast) в правом верхнем углу. Уведомления автоматически исчезают через несколько секунд и имеют цветовую индикацию (успех/ошибка).
- **Валидация формы проекта** — перед сохранением обязательные поля (название и статус) проверяются на заполненность. Поля с ошибками подсвечиваются.
- **Индикатор загрузки** — при загрузке файлов отображается вращающийся спиннер рядом с кнопкой загрузки.
- **Доступность** — добавлена ссылка «пропустить содержимое» для пользователей, перемещающихся с клавиатурой, фокусные обводки для интерактивных элементов и aria‑атрибуты для кнопок закрытия. Эти улучшения повышают доступность интерфейса.

## Быстрый старт

1. Установите зависимости:

   ```bash
   npm install
   ```

2. Запустите сервер и клиент одновременно:

   ```bash
   npm start
   ```

   По умолчанию сервер Express доступен на порту `5000`, а клиент React — на `3000`. Сервер хранит файлы в директории `server/uploads` и данные проекта в `server/project.json`.

3. Откройте приложение в браузере: `http://localhost:3000`.

## Настройка API

Клиентское приложение использует переменную окружения `REACT_APP_API_URL` для определения базового URL API. Если эта переменная не задана, используется `http://localhost:5000/api`. Чтобы сменить адрес API (например, при развёртывании сервера на другой машине), создайте файл `.env` в корне проекта и укажите:

```
REACT_APP_API_URL=http://your-server:5000/api
```

## Структура проекта

- **server/** — исходный код Express‑сервера. Маршруты API находятся в `index.js`, данные проекта — в `project.json`, настройки фона — в `settings.json`, а загруженные файлы — в папке `uploads`.
- **src/** — исходный код React‑приложения. Компонент `App.js` содержит основную логику, `ProjectDetails.js` отвечает за карточку проекта и выбор фона, а компонент `Card.js` используется для отображения файлов.

## Улучшения по сравнению с исходным прототипом

В исходном архиве интерфейс представлял собой файловый менеджер. В этой версии реализованы дополнительные требования кейса:

- Добавлена карточка строительного проекта с возможностью редактирования статуса, адреса, дат и ответственного.
- Перенесён выбор и загрузка фона в карточку проекта; вкладка «Фон» удалена, что делает основной сценарий более очевидным.
- Используются переменные окружения для задания URL API, что упрощает развёртывание.
- Добавлен `README.md` с инструкциями по запуску и описанием возможностей.
